# RAFIQ Localization Rules V1

Status: Build Gate
Last updated: 2026-06-09

## Purpose

This document clarifies what the AI may and may not translate.

## Launch Language Decision

Default build assumption:

- primary UI language: English
- supported launch fallback: Malay if approved translation sources are available
- Arabic is shown for Quran/hadith where source data supports it

This can be changed before implementation, but the source approval rules remain.

## Translation Rules

| Content | Translation Rule |
| --- | --- |
| Quran | Never translated by AI. Use approved translation records only. |
| Hadith | Prefer approved translation records. If AI translation is ever used, label it clearly and do not use it as a source of religious proof. |
| Tafsir | Use approved source translation or approved editorial summary. AI may simplify approved tafsir, but must cite the source. |
| Reflection | AI may generate in supported UI language. |
| Action | AI may generate in supported UI language if safety gates pass. |
| UI copy | Product team may translate or use AI-assisted translation with human review. |

## Fallback Order

1. User's selected language if approved source exists.
2. English approved source.
3. Show source unavailable message.

Never silently substitute an AI-generated Quran translation.

## User-Facing Labels

Use labels such as:

- `Translation: Saheeh International`
- `Malay translation source: [approved source name]`
- `Reflection generated by Rafiq`
- `Summary based on [tafsir source]`
